# ADR 0006：Phase 1 报告格式优先使用 Markdown、CSV、JSON

## 状态

已接受，作为 Phase 1 报告输出方向。

## 背景

第一阶段需要让用户能看懂、保存、分享和二次处理结果。

不同场景需要不同格式：

- 运维自己看：CLI 输出。
- 归档和分享：Markdown。
- Excel 处理：CSV。
- 脚本或后续系统处理：JSON。
- 未来 Web 展示：HTML 或 API。

## 决策

Phase 1 优先支持：

- CLI 简洁输出。
- Markdown 报告。
- CSV 导出。
- JSON 输出。

HTML 报告可以后置。

## 备选方案

### 直接做 HTML 报告

优点：

- 展示效果好。
- 接近未来 Web 方向。

缺点：

- 第一阶段会增加模板和样式复杂度。
- 容易把精力放在展示而不是数据结构。

### 只做 CLI 输出

优点：

- 实现最快。

缺点：

- 不适合归档。
- 不适合给领导或同事分享。
- AI 和后续 Web 复用价值低。

## 后果

正面影响：

- Markdown 易读、易归档。
- CSV 方便导入 Excel。
- JSON 方便脚本和测试。
- CLI 保持现场使用效率。

负面影响：

- 需要维护多种输出格式。
- 报告模块必须和数据模型保持一致。

## 执行要求

- 报告必须从结构化结果生成。
- CLI 输出不能成为唯一数据来源。
- Markdown 报告要包含执行时间、目标、摘要、异常、证据和建议。
- CSV 主要用于资产和巡检结果表格。
- JSON 输出应尽量接近内部结构化结果。
