# 报告中心模块

## 模块职责

报告中心负责把结构化结果转成可读、可分享、可归档的内容。

报告不是简单美化输出，它是运维工作的交付物。中小企业里，报告经常用于：

- 给领导说明环境状态。
- 给业务部门说明故障原因。
- 给自己留下历史记录。
- 给交接人员提供依据。
- 给 AI 总结和复盘提供上下文。

## 不负责什么

本模块不负责：

- 执行探测。
- 判断故障原因。
- 发送通知。
- 保存资产主数据。
- 直接调用外部协议。

它只消费结构化结果。

## 输入

输入可以包括：

- 资产结果。
- 巡检结果。
- 诊断结果。
- 安全发现。
- 自动化执行记录。
- 报告模板。
- 输出格式配置。

## 输出

输出可以包括：

- CLI 表格。
- Markdown 报告。
- HTML 报告。
- CSV 导出。
- JSON 结果。
- 巡检日报。
- 周报。
- 故障复盘。

## 依赖模块

依赖：

- 数据存储。
- 配置中心。
- 资产与拓扑。
- 网络与服务巡检。
- 场景化故障诊断。
- 轻量安全与合规。

被依赖：

- CLI。
- Web Console。
- AI 运维助手。
- 通知中心。

## 第一阶段范围

第一阶段建议支持：

- CLI 简洁输出。
- Markdown 报告。
- CSV 导出。
- 巡检报告。
- 资产报告。

报告内容应尽量包含：

- 执行时间。
- 检查目标。
- 总体结论。
- 异常清单。
- 证据。
- 建议下一步。

## 未来扩展

后续可扩展：

- HTML 可视化报告。
- Web 报告查看。
- 定时日报和周报。
- 故障复盘模板。
- AI 改写成领导版摘要。
- 报告版本管理。

## 验收标准

第一阶段完成时，应满足：

- 报告从结构化结果生成。
- 报告能区分正常和异常。
- 报告能列出证据。
- 报告可保存到本地文件。
- 报告模块不执行检查。
