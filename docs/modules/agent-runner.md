# Agent Runner 模块

## 模块职责

Agent Runner 是未来模块，用于在权限、审批、审计和风险控制下执行预定义运维工作流。

它不是“让 AI 随便操作系统”，而是“让 Agent 在受控流程里调用已有能力”。

## 现实场景

未来可能希望 Agent 自动处理：

- 收到“上不了网”后执行只读诊断流程。
- 根据巡检异常自动补充检查。
- 生成故障复盘草稿。
- 对低风险动作提出执行建议。
- 对高风险动作请求人工审批。

如果没有 Agent Runner 边界，AI 很容易变成不可审计的黑箱自动化。

## 不负责什么

第一阶段不实现 Agent Runner。

本模块未来也不负责：

- 无限权限执行。
- 直接绕过审批。
- 自己实现底层探测。
- 隐藏执行步骤。
- 直接处理明文凭据。

## 输入

未来输入可以包括：

- 工作流定义。
- 用户目标。
- 当前上下文。
- 可用工具列表。
- 风险策略。
- 权限信息。
- 审批状态。

## 输出

未来输出包括：

- 执行计划。
- 执行步骤。
- 每步结果。
- 暂停原因。
- 审批请求。
- 最终报告。
- 审计记录。

## 依赖模块

依赖：

- AI 运维助手。
- 自动化动作。
- 场景化故障诊断。
- 权限与审计。
- 数据存储。
- 日志与观测性。

被依赖：

- Web Console。
- 未来 API。

## 第一阶段范围

第一阶段只做架构预留：

- 不实现 Agent。
- 在文档中明确风险边界。
- 在自动化动作中保留风险等级。
- 在权限审计中保留执行记录模型。
- 在 AI 模块中明确“不直接执行”的原则。

## 未来扩展

后续可扩展：

- 只读诊断 Agent。
- 报告生成 Agent。
- 审批型修复 Agent。
- 多步骤 Runbook Agent。
- Web Console 中的 Agent 执行记录。

## 验收标准

第一阶段验收标准：

- 总架构没有阻碍未来 Agent。
- 自动化动作已有风险分级。
- 审计模型能记录动作。
- AI 模块没有默认执行权限。
