# 凭据与密钥管理模块

## 模块职责

凭据与密钥管理模块负责管理凭据引用和敏感信息处理规则。

它的核心目标是避免密码、Token、私钥散落在配置、日志、报告和脚本里。

## 现实场景

中小企业工具脚本常见问题：

- 密码直接写在脚本里。
- Token 直接写在配置文件里。
- 日志输出了完整命令和密码。
- 报告里带了敏感 URL 或认证信息。
- 多个人复制同一份脚本，凭据到处扩散。

这个模块需要从第一阶段就定规则，即使完整凭据库以后再做。

## 不负责什么

第一阶段不负责：

- 完整企业级 Vault。
- 多用户凭据共享。
- 复杂密钥轮换。
- 明文密码存储。

## 输入

输入可以包括：

- 凭据引用 ID。
- 环境变量引用。
- 本机安全存储引用。
- 未来 Vault 引用。
- 脱敏规则。

## 输出

输出包括：

- 解析后的凭据引用。
- 脱敏后的日志内容。
- 凭据缺失错误。
- 访问权限检查结果。

## 依赖模块

依赖：

- 配置中心。
- 权限与审计。
- 日志与观测性。

被依赖：

- 插件与探针系统。
- 自动化动作。
- 通知中心。
- Agent Runner。

## 第一阶段范围

第一阶段建议做到：

- 配置文件中使用凭据引用，不鼓励明文密码。
- 日志和报告默认脱敏。
- 文档明确禁止硬编码凭据。
- 为未来 OS Keychain 或 Vault 预留接口。

## 未来扩展

后续可扩展：

- Windows Credential Manager。
- macOS Keychain。
- Linux Secret Service。
- HashiCorp Vault。
- 云厂商 Secret Manager。
- 凭据权限控制。

## 验收标准

第一阶段完成时，应满足：

- 文档明确敏感信息规则。
- 日志不输出明文密码。
- 报告不输出明文 Token。
- 代码不鼓励硬编码凭据。
- Adapter 使用凭据引用而不是直接明文。
