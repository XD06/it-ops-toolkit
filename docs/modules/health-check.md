# 网络与服务巡检模块

## 模块职责

网络与服务巡检模块负责回答：关键网络路径和业务服务当前是否正常。

它不是发现“有什么”的模块，而是检查“是否健康”的模块。

## 现实场景

中小企业常见巡检对象包括：

- 默认网关。
- DNS 服务器。
- DHCP 服务。
- 内网业务系统。
- 文件服务器。
- 远程桌面服务。
- VPN。
- 打印服务。
- 互联网出口。
- 常用 SaaS 或云服务。

很多故障不是完全不可用，而是 DNS 慢、端口偶发超时、HTTP 状态异常、证书快过期、网关延迟高。巡检模块需要把这些情况结构化记录下来。

## 不负责什么

本模块不负责：

- 维护资产归属。
- 生成复杂故障结论。
- 执行修复动作。
- 判断所有安全风险。
- 发送通知。

它提供检查结果，诊断、安全、报告、通知模块再消费这些结果。

## 输入

输入可以包括：

- 目标列表。
- 服务定义。
- 巡检配置。
- 超时设置。
- 重试策略。
- 检查类型：Ping、DNS、TCP、HTTP。
- CSV 批量目标文件，用于批量 TCP matrix。
- CSV 批量目标文件，用于批量 HTTP matrix。

## 输出

输出应是结构化巡检结果，例如：

- 目标。
- 检查类型。
- 状态：正常、异常、超时、部分异常。
- 响应时间。
- 错误原因。
- DNS 解析结果。
- TCP 连接状态。
- HTTP 状态码。
- 证书信息。
- 检查时间。
- 证据。

## 依赖模块

依赖：

- 配置中心。
- 插件与探针系统。
- 数据存储。
- 日志与观测性。

被依赖：

- 场景化故障诊断。
- 报告中心。
- AI 运维助手。
- 通知中心。
- 轻量安全与合规。

## 第一阶段范围

第一阶段巡检能力：

- Ping 网关或指定目标。
- DNS 解析检查。
- TCP 端口检查。
- HTTP 状态检查。
- 基础超时和失败记录。
- 巡检结果保存。
- 巡检报告输出。
- 批量 TCP matrix，只做 TCP 可达性检查。
- 批量 HTTP matrix，只做 HTTP/HTTPS 可达性检查。

## 未来扩展

后续可扩展：

- 定时巡检。
- 趋势统计。
- SLA 简化统计。
- 抖动和丢包趋势统计（Ping Probe 已支持单次丢包率和 RTT 解析，趋势分析待实现）。
- 证书链检查。
- NTP 检查。
- DHCP 检查。
- 多出口对比。
- Web 控制台健康看板。

## 验收标准

第一阶段完成时，应满足：

- 能读取配置中的巡检目标。
- 能批量执行 Ping、DNS、TCP、HTTP 检查。
- 能批量执行 TCP matrix 检查。
- 能批量执行 HTTP matrix 检查。
- 能保存结构化巡检结果。
- 能生成一份巡检报告。
- 单个目标失败不影响其他目标执行。
- 巡检逻辑不写在 CLI 内部。
