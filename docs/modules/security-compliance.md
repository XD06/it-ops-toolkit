# 轻量安全与合规模块

## 模块职责

轻量安全与合规模块负责发现中小企业 IT 环境中常见、明显、可操作的风险。

它不是完整安全平台，而是把日常最容易忽略的基础风险纳入运维工具箱。

## 现实场景

常见风险包括：

- 未知设备接入办公网。
- 内网设备开放高风险端口。
- 数据库端口暴露在不该出现的位置。
- 远程桌面端口长期开放。
- 证书即将过期。
- 关键设备没有明确负责人。
- 补丁状态不清楚。
- 备份是否可用没人确认。

中小企业通常没有完整 SIEM 或漏洞管理平台，所以轻量检查很有现实价值。

## 不负责什么

本模块不负责：

- 漏洞利用。
- 密码爆破。
- 完整渗透测试。
- EDR 功能。
- SIEM 关联分析。
- 自动修复风险。

## 输入

输入可以包括：

- 资产记录。
- 开放端口结果。
- 巡检结果。
- 证书信息。
- 安全规则配置。
- 历史结果。

## 输出

输出应包括：

- 风险标题。
- 风险等级。
- 受影响资产。
- 证据。
- 建议处理方式。
- 是否需要人工确认。
- 是否适合通知。

## 依赖模块

依赖：

- 资产与拓扑。
- 网络与服务巡检。
- 配置中心。
- 数据存储。

被依赖：

- 报告中心。
- AI 运维助手。
- 通知中心。
- Agent Runner。

## 第一阶段范围

第一阶段建议支持：

- 高风险端口检查。
- 未知设备提示。
- 新开放端口提示。
- 证书过期检查。
- 基础风险报告。

高风险端口示例：

- 3389 RDP。
- 445 SMB。
- 3306 MySQL。
- 1433 SQL Server。
- 6379 Redis。
- 22 SSH。

风险判断要结合配置，不应简单认为所有开放端口都是错误。

## 未来扩展

后续可扩展：

- Windows 防火墙状态。
- 杀毒软件状态。
- 补丁状态。
- 备份状态。
- 账号锁定和异常登录分析。
- 基线检查。
- 合规报告。

## 验收标准

第一阶段完成时，应满足：

- 能基于资产和端口结果产生风险发现。
- 每个风险都有证据。
- 风险等级可配置。
- 报告能展示风险清单。
- 不执行攻击性测试。
