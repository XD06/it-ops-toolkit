# 数据存储模块

## 模块职责

数据存储模块负责保存平台结构化数据。

它不是业务规则模块，只提供数据读写能力和稳定的数据模型。

## 现实场景

如果不保存历史，系统只能回答“现在是什么情况”，不能回答：

- 这个设备什么时候第一次出现。
- 这个 IP 最近是否换过设备。
- 某服务从什么时候开始异常。
- 上周巡检和今天有什么变化。
- 哪次任务生成了这份报告。
- AI 总结基于哪些结果。

所以数据存储是 Web、AI、报告和审计的基础。

## 不负责什么

本模块不负责：

- 判断风险。
- 执行探测。
- 生成报告内容。
- 发送通知。
- 保存明文敏感凭据。

## 输入

输入可以包括：

- 资产记录。
- ProbeResult。
- TaskRun。
- Finding。
- Report。
- Event。
- KnowledgeEntry。

## 输出

输出包括：

- 查询结果。
- 保存结果。
- 数据引用 ID。
- 错误信息。

## 依赖模块

依赖：

- 配置中心。
- 日志与观测性。

被依赖：

- 几乎所有业务模块。
- Web Console。
- AI 运维助手。
- 报告中心。
- 权限与审计。

## 第一阶段范围

第一阶段可以使用本地文件或轻量数据库。

需要保存：

- 资产。
- 任务记录。
- 探测结果。
- 报告元数据。

具体存储技术在实现计划中决定。

## 未来扩展

后续可扩展：

- SQLite。
- PostgreSQL。
- 数据迁移。
- 数据保留策略。
- 历史趋势。
- Web 查询接口。

## 验收标准

第一阶段完成时，应满足：

- 能保存结构化结果。
- 能按任务查询结果。
- 能按资产查询历史。
- 能被报告模块读取。
- 存储接口不绑定某个 UI。
